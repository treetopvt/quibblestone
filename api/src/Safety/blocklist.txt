# ----------------------------------------------------------------------------
#  blocklist.txt - the bundled baseline safety blocklist for QuibbleStone.
#
#  This file is embedded into the API assembly (see QuibbleStone.Api.csproj,
#  <EmbeddedResource>) and loaded once by ContentSafetyFilter at startup. It is
#  the slice-1 baseline (CLAUDE.md section 7): a SOLID starting set, deliberately
#  NOT locale-complete or exhaustive coverage (that is out of scope for the slice;
#  AI / remote moderation is parked in the backlog, README section 12).
#
#  Format:
#    - One blocked term per line, lower-case.
#    - Blank lines are ignored.
#    - Lines starting with '#' are comments (like this header) and are ignored.
#    - Matching is done on normalized text (lower-cased, common leet-speak
#      substitutions folded, runs of repeats collapsed) - see ContentSafetyFilter.
#      So you do NOT need to list every spelling variant here; the normalizer
#      handles the obvious ones (e.g. "@" -> "a", "0" -> "o", "$" -> "s").
#
#  To extend coverage: add terms here, lower-case, one per line. No code change
#  is needed - the filter reads whatever ships in this resource. Keep it tasteful
#  and child-appropriate: this is the floor that keeps the game safe to hand to a
#  kid, not a comprehensive linguistic database.
#
#  No em dashes anywhere (project prose rule).
# ----------------------------------------------------------------------------

# --- Compound / fused profanity ---
# The matcher is WHOLE-WORD by design (it never substring-scans, to avoid the
# Scunthorpe problem - "ass" in "class", "cum" in "cucumber", "shit" in the
# mushroom "shiitake", "cunt" in "Scunthorpe"). A consequence is that profanity
# FUSED into one token ("shithead") is only caught if the whole compound is
# listed here. So the common, unambiguous compounds are listed explicitly - each
# is obscene as a whole word, so it adds zero false-positive risk. This is the
# curated middle ground; broad generative coverage is the parked AI-moderation
# work (README section 12), not a substring rule.
